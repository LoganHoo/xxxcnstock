# XCNStock 项目说明文档

## 项目概述
- **项目名称**: XCNStock A股量化分析系统
- **项目边界**: 本项目仅做流水线的SOP，不做实盘交易，可以做回测
- **技术栈**: Python 3.11+ / Flask / SQLAlchemy / MySQL 5.0+ / Redis / Next.js 16+
- **创建日期**: 2026-03-24
- **项目方向**: A股市场分析（用户确认：2026-03-24）

## 数据存储边界

### ✅ 存储的数据
- **历史行情数据**：A股历史K线数据（日线级别）
  - 数据范围：最近3年历史数据
  - 更新频率：每日收盘后更新一次
  - 存储格式：Parquet列式存储
  - 数据字段：股票代码、交易日期、开盘价、收盘价、最高价、最低价、成交量

### ❌ 不存储的数据
- **实时行情数据**：不存储实时行情
  - 实时行情通过API实时查询
  - 不占用存储空间
  - 保持数据的时效性

### 数据更新策略
- **每日更新**：收盘后自动更新历史数据
- **增量更新**：只更新新增的交易日数据
- **数据完整性**：确保历史数据的完整性和准确性

## 数据处理技术栈

### 核心技术
- **数据格式**: Parquet - 高效列式存储格式
- **数据处理**: Polars - 高性能DataFrame库
- **数据查询**: DuckDB - 嵌入式分析数据库

### 性能优势
- **Parquet**: 
  - 压缩率高，文件小（平均20KB/股票）
  - 列式存储，查询速度快
  - 总数据量：121MB（5079只股票，324万条记录）

- **Polars**:
  - 比Pandas快10-100倍
  - 内存效率高，支持懒加载
  - 多线程和SIMD优化
  - 适用场景：大规模数据清洗、转换、聚合

- **DuckDB**:
  - 支持SQL查询Parquet文件
  - 零配置，无需服务器
  - 支持窗口函数、聚合、JOIN等复杂查询
  - 适用场景：复杂SQL分析、多表关联、数据挖掘

### 数据存储结构
```
data/
├── kline/                    # K线数据目录
│   ├── 000001.parquet       # 每个股票一个文件
│   ├── 000002.parquet
│   └── ...
├── stock_list.parquet       # 股票列表

```

### 使用示例

#### 1. 使用Polars读取数据
```python
import polars as pl

# 读取单个股票数据
df = pl.read_parquet('data/kline/000001.parquet')

# 读取所有股票数据（懒加载）
df_all = pl.scan_parquet('data/kline/*.parquet')

# 计算平均收盘价
avg_close = df_all.group_by('code').agg(
    pl.col('close').mean().alias('avg_close')
).collect()
```

#### 2. 使用DuckDB查询数据
```python
import duckdb

# 查询单个股票
result = duckdb.query("""
    SELECT * FROM 'data/kline/000001.parquet'
    WHERE trade_date >= '2024-01-01'
    ORDER BY trade_date
""").df()

# 查询所有股票的最新价格
result = duckdb.query("""
    SELECT code, trade_date, close
    FROM 'data/kline/*.parquet'
    WHERE (code, trade_date) IN (
        SELECT code, MAX(trade_date)
        FROM 'data/kline/*.parquet'
        GROUP BY code
    )
    ORDER BY code
""").df()

# 计算每只股票的涨跌幅
result = duckdb.query("""
    SELECT 
        code,
        trade_date,
        close,
        LAG(close) OVER (PARTITION BY code ORDER BY trade_date) as prev_close,
        (close - LAG(close) OVER (PARTITION BY code ORDER BY trade_date)) / 
        LAG(close) OVER (PARTITION BY code ORDER BY trade_date) * 100 as pct_change
    FROM 'data/kline/000001.parquet'
    ORDER BY trade_date DESC
    LIMIT 10
""").df()
```

#### 3. 使用Pandas读取数据（兼容性）
```python
import pandas as pd

# 读取单个股票数据
df = pd.read_parquet('data/kline/000001.parquet')

# 查看数据
print(df.head())
print(df.info())
```

- `market`: 所属市场（上海/深圳）
- `status`: 状态（active/delisted/suspended）

**K线数据 (data/kline/*.parquet)**:
- `code`: 股票代码
- `trade_date`: 交易日期
- `open`: 开盘价
- `close`: 收盘价
- `high`: 最高价
- `low`: 最低价
- `volume`: 成交量
