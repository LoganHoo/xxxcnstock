## Phase Goal

**As a** system operator, **I want to** monitor the scheduler and have it automatically handle common failure scenarios (Redis outages, persistently failing tasks, market holidays), **so that** the scheduler runs reliably without requiring constant human oversight.

## Acceptance Criteria

- [ ] When Redis is down, tasks execute without distributed locking (degraded mode, logged warning)
- [ ] Every task execution is recorded in SQLite with start time, end time, status, and error details
- [ ] Circuit breaker opens after N consecutive failures, skips the task, and auto-recovers after timeout
- [ ] Tasks with day_type=weekday do not run on weekends or holidays
- [ ] Flask API process serves /health, /tasks, /stats by reading shared SQLite database
- [ ] Engine and API processes are fully independent (API crash does not affect engine)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Redis -> scheduler | External dependency; connection may fail or be slow |
| API process -> SQLite | Shared database file with engine process; WAL mode prevents corruption |
| API endpoints -> network | Flask serves status data; internal network only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Denial of Service | lock_manager.py | mitigate | Redis connection timeout (3s connect, 5s read) prevents indefinite blocking. Fresh client per attempt avoids poisoned pools. |
| T-01-06 | Tampering | history.py | accept | SQLite DB on local filesystem; internal service, no remote access. WAL mode prevents corruption from concurrent access. |
| T-01-07 | Information Disclosure | api/routes.py | accept | API returns task names, statuses, durations -- no credentials or sensitive data. Internal network only. |
| T-01-08 | Denial of Service | circuit_breaker.py | mitigate | Circuit breaker itself prevents cascading failures by stopping repeated execution of failing tasks. State files are simple JSON with atomic writes. |
</threat_model>
