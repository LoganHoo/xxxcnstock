# 量化交易系统 V2 - 架构文档

## 🎯 核心改进总结

针对专业分析提出的6大问题，V2版本进行了系统性重构：

| 问题 | 解决方案 | 实现模块 |
|:---|:---|:---|
| 1. 数据采集与清洗顺序错误 | ETL规范：Extract → Transform → Load | `ETLProcessor` |
| 2. 缺乏回测引擎 | 完整回测框架 + IC值计算 | `BacktestEngine` |
| 3. 9:26性能瓶颈 | 优先级队列 + 极速模式 | `PriorityDataCollector`, `FastPreMarketSystem` |
| 4. 缺乏风险控制 | 6大风控规则 + 独立校验 | `RiskManager` |
| 5. 选股与盘中脱节 | 动态头寸管理 | `PositionManager` |
| 6. 技术实现缺失 | 优先级队列、除权除息、VWAP/TWAP | 各模块具体实现 |

## 📦 模块详解

### 1. 优先级数据采集器 (PriorityDataCollector)

**解决的问题**: 核心行情数据被非核心资讯堵塞

```python
# 数据优先级定义
CRITICAL = 1    # 核心行情（必须优先）
HIGH = 2        # K线、指数
MEDIUM = 3      # 基本面
LOW = 4         # CCTV、外盘
BACKGROUND = 5  # 历史补采
```

**工作流程**:
1. 任务按优先级入队（堆队列）
2. 同优先级任务并发执行
3. 带指数退避的重试机制
4. 超时控制（默认30秒）

---

### 2. ETL处理器 (ETLProcessor)

**解决的问题**: 数据清洗顺序错误

**正确流程**: Extract → Transform → Load → Quality Check

```
原始数据 (Raw)
    ↓
清洗 (Clean) - 去除异常值、去重、排序
    ↓
除权除息 (Adjust) - 应用复权因子
    ↓
标准化 (Normalize) - 统一列名、计算派生字段
    ↓
质量标记 (Tag) - 标记质量等级、缺失值
    ↓
质量检查 (Quality Check) - 完整性、一致性、及时性
    ↓
加载 (Load) - 存入数据仓库
```

**除权除息处理**:
```python
# 获取复权因子
adj_factor = get_adjustment_factor(code)

# 应用复权
df['close_adj'] = df['close'] * df['adj_factor']
df['open_adj'] = df['open'] * df['adj_factor']
# ...
```

### 3. 回测引擎 (BacktestEngine)

**解决的问题**: 缺乏因子有效性验证

**核心指标**:
| 指标 | 说明 | 计算方式 |
|:---|:---|:---|
| IC值 | 信息系数，因子预测能力 | corr(因子值, 未来收益) |
| IR值 | 信息比率，因子稳定性 | mean(IC) / std(IC) |
| 年化收益 | 策略收益能力 | (1+总收益)^(365/天数) - 1 |
| 最大回撤 | 风险控制能力 | max(峰值-谷值)/峰值 |
| 夏普比率 | 风险调整收益 | 超额收益 / 收益波动率 |
| 胜率 | 交易成功率 | 盈利次数 / 总次数 |

**回测流程**:
```
加载历史数据
    ↓
计算因子值
    ↓
计算IC值（验证因子有效性）
    ↓
模拟交易（按信号）
    ↓
计算回测指标
    ↓
生成回测报告
```

### 4. 风险管理器 (RiskManager)

**解决的问题**: 缺乏风险控制

**6大风控规则**:

| 规则 | 限制 | 严重级别 | 动作 |
|:---|:---|:---:|:---:|
| 单股持仓上限 | ≤10% | error | block |
| 行业集中度 | ≤30% | error | block |
| 单日回撤 | ≤5% | error | block |
| 总回撤 | ≤20% | error | block |
| 流动性检查 | 成交额>1000万 | warning | alert |
| 波动率检查 | ATR<5% | warning | alert |

**使用方式**:
```python
# 检查订单
results = risk_manager.check_order(order, portfolio)

# 判断能否执行
can_execute, reasons = risk_manager.can_execute(order, portfolio)
if not can_execute:
    print(f"风控拦截: {reasons}")
```

### 5. 极速盘前系统 (FastPreMarketSystem)

**解决的问题**: 9:26报告滞后

**优化策略**:
1. **只加载核心数据** - 跳过CCTV、基本面等非关键数据
2. **跳过非关键检查** - 只做核心质量检查
3. **异步并行处理** - 多线程并发
4. **60秒超时控制** - 确保及时输出

**执行时间分配**:
| 步骤 | 时间预算 | 说明 |
|:---|:---:|:---|
| 数据采集 | 15秒 | 只采集合竞价数据 |
| 涨停分析 | 20秒 | 快速筛选涨停股 |
| 开板预测 | 15秒 | 简化模型预测 |
| 生成报告 | 10秒 | 输出推荐列表 |
| **总计** | **60秒** | 硬截止时间 |

### 6. 订单执行算法

**解决的问题**: 大单直接砸入市场产生滑点

#### VWAP算法 (成交量加权平均价格)

```python
# 将大单拆分为多个小单
# 按历史成交量分布执行

子订单1: 9:31 执行 10% (历史该时段成交量占比)
子订单2: 9:32 执行 15%
子订单3: 9:33 执行 12%
...
```

#### TWAP算法 (时间加权平均价格)

```python
# 将大单按时间均匀拆分
# 每N秒执行一部分

子订单1: 9:31:00 执行 100股
子订单2: 9:32:00 执行 100股
子订单3: 9:33:00 执行 100股
...
```

### 7. 动态头寸管理 (PositionManager)

**解决的问题**: 选股评分与盘中监控脱节

**动态调整逻辑**:
```python
# 根据市场波动率调整权重
if 波动率 > 3%:    # 高波动
    仓位调整 = 0.7   # 降低至70%
elif 波动率 < 1%:  # 低波动
    仓位调整 = 1.2   # 增加至120%
else:
    仓位调整 = 1.0   # 维持100%

# 应用到所有选股权重
for code, score in selection_scores.items():
    target_weights[code] = score * 仓位调整
```

**再平衡触发**:
- 目标权重与当前权重差异 > 1%
- 市场波动率变化 > 0.5%
- 新的选股信号产生

## 🎓 设计原则

1. **数据优先**: ETL规范确保数据质量
2. **验证先行**: 回测验证因子有效性
3. **风控独立**: 所有订单必须经过风控
4. **性能保障**: 关键路径限时完成
5. **动态适应**: 根据市场状态调整策略
