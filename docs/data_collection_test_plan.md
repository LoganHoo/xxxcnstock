# 数据采集测试计划

## 测试目标

验证所有数据类型的采集功能，确保：
1. 可采集1年以上历史数据
2. 支持每日增量更新
3. 支持断点续传
4. 数据质量符合要求

---

## 一、K线数据采集测试

### 1.1 全量采集测试（1年以上）

**测试脚本**: `scripts/test_kline_full_collection.py`

**测试内容**:
| 测试项 | 预期结果 | 验证方法 |
|--------|----------|----------|
| 采集1年历史数据 | 成功获取约250个交易日 | 检查数据行数 |
| 采集2年历史数据 | 成功获取约500个交易日 | 检查数据行数 |
| 采集3年历史数据 | 成功获取约750个交易日 | 检查数据行数 |
| 多股票并行采集 | 10只股票同时采集成功 | 检查所有文件 |

**测试股票**: 600000、000001、300001、688001、000002

### 1.2 增量更新测试

**测试脚本**: `scripts/test_kline_incremental.py`

**测试内容**:
| 测试项 | 预期结果 | 验证方法 |
|--------|----------|----------|
| 已有数据增量更新 | 只采集新数据 | 对比文件修改时间 |
| 最新数据已存在 | 跳过采集 | 检查日志输出 |
| 部分缺失数据 | 补全缺失部分 | 检查数据连续性 |

### 1.3 断点续传测试

**测试脚本**: `scripts/test_kline_resume.py`

**测试内容**:
| 测试项 | 预期结果 | 验证方法 |
|--------|----------|----------|
| 中断后重新采集 | 从断点继续 | 检查进度文件 |
| 已完成的股票跳过 | 不重复采集 | 检查处理时间 |
| 部分完成的股票 | 继续未完成部分 | 检查数据完整性 |

## 七、测试执行计划

### 7.1 执行顺序

```
第1阶段: K线数据采集测试
  - 1.1 全量采集测试
  - 1.2 增量更新测试
  - 1.3 断点续传测试

第2阶段: 财务数据采集测试
  - 2.1 全量采集测试
  - 2.2 增量更新测试

第3阶段: 市场行为数据采集测试
  - 3.1 龙虎榜采集测试
  - 3.2 资金流向采集测试

第4阶段: 公告数据采集测试
  - 4.1 公司公告采集测试

第5阶段: 断点续传机制测试
  - 5.1 进度保存测试
  - 5.2 异常恢复测试

第6阶段: 数据质量验证
  - 6.1 完整性检查
  - 6.2 一致性检查
```

### 7.2 测试环境

- **数据源**: Baostock（主）、腾讯（备）
- **测试股票**: 5只代表性股票
- **时间范围**: 1年以上历史数据
- **网络**: 正常网络环境

### 7.3 通过标准

| 测试类型 | 通过率要求 |
|----------|-----------|
| K线采集 | 100% |
| 财务采集 | 100% |
| 市场行为采集 | >=90% |
| 公告采集 | >=90% |
| 断点续传 | 100% |
| 数据质量 | >=95% |
