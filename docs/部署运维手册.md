# 部署运维手册

本文档提供数据服务扩展项目的部署、运维和故障处理指南。

## 一、环境要求

### 1.1 硬件要求

| 环境 | CPU | 内存 | 存储 | 网络 |
|------|-----|------|------|------|
| 开发环境 | 4核 | 8GB | 50GB | 10Mbps |
| 测试环境 | 4核 | 8GB | 100GB | 10Mbps |
| 生产环境 | 8核+ | 16GB+ | 500GB+ | 100Mbps |

### 1.2 软件要求

- **操作系统**: Linux (Ubuntu 20.04+ / CentOS 8+) / macOS / Windows WSL2
- **Python**: 3.11+
- **依赖包**: 见 requirements.txt

### 1.3 外部依赖

- **Tushare Pro API**: 需要有效Token
- **AKShare**: 备用数据源
- **网络**: 可访问外网获取数据

## 二、部署流程

### 2.1 首次部署

```bash
# 1. 克隆代码仓库
git clone <repository_url>
cd xxxcnstock

# 2. 创建虚拟环境
python3.11 -m venv venv
source venv/bin/activate  # Linux/macOS
# 或 venv\Scripts\activate  # Windows

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，配置以下参数：
# - TUSHARE_TOKEN=your_token_here
# - DATA_DIR=/path/to/data
# - LOG_DIR=/path/to/logs

# 5. 创建目录结构
mkdir -p data/{financial,market_behavior,announcements,quality_reports}
mkdir -p logs/system
mkdir -p config/filters

# 6. 初始化配置文件
python scripts/init_config.py

# 7. 验证安装
python -c "from services.data_service.unified_data_service import UnifiedDataService; print('安装成功')"
```

### 2.2 Docker部署

```dockerfile
# Dockerfile
FROM python:3.11-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    gcc \
    g++ \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制代码
COPY . .

# 创建数据目录
RUN mkdir -p data logs

# 设置环境变量
ENV PYTHONPATH=/app
ENV DATA_DIR=/app/data
ENV LOG_DIR=/app/logs

# 启动命令
CMD ["python", "-m", "services.data_service.tasks.daily_update_task"]
```

```yaml
# docker-compose.yml
version: '3.8'

services:
  data-service:
    build: .
    container_name: xxxcnstock-data-service
    volumes:
      - ./data:/app/data
      - ./logs:/app/logs
      - ./config:/app/config
    environment:
      - TUSHARE_TOKEN=${TUSHARE_TOKEN}
      - LOG_LEVEL=INFO
    restart: unless-stopped
    
  # 可选：Redis缓存
  redis:
    image: redis:7-alpine
    container_name: xxxcnstock-redis
    volumes:
      - redis_data:/data
    restart: unless-stopped

volumes:
  redis_data:
```

```bash
# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f data-service

# 停止服务
docker-compose down
```

## 四、定时任务配置

### 4.1 Crontab配置

```bash
# 编辑crontab
crontab -e

# 添加以下任务
# 数据预热 (工作日 8:30)
30 8 * * 1-5 cd /path/to/xxxcnstock && /path/to/venv/bin/python -m services.data_service.tasks.data_preheat_task >> logs/cron.log 2>&1

# 收盘后更新 (工作日 17:00)
0 17 * * 1-5 cd /path/to/xxxcnstock && /path/to/venv/bin/python -m services.data_service.tasks.daily_update_task >> logs/cron.log 2>&1

# 数据质量检查 (每天 20:00)
0 20 * * * cd /path/to/xxxcnstock && /path/to/venv/bin/python -m services.data_service.quality.data_quality_monitor --check >> logs/cron.log 2>&1

# 生成监控报告 (每天 9:00)
0 9 * * * cd /path/to/xxxcnstock && /path/to/venv/bin/python -m services.data_service.monitoring.data_service_dashboard --report --save >> logs/cron.log 2>&1

# 增量更新 (每周一 1:00)
0 1 * * 1 cd /path/to/xxxcnstock && /path/to/venv/bin/python -m services.data_service.tasks.incremental_update_task >> logs/cron.log 2>&1
```

### 4.2 使用APScheduler

```python
# 使用Python调度器替代crontab
from apscheduler.schedulers.background import BackgroundScheduler
from services.data_service.tasks import DailyUpdateTask, DataPreheatingTask

scheduler = BackgroundScheduler()

# 数据预热
scheduler.add_job(
    DataPreheatingTask().run_preheating,
    'cron',
    hour=8,
    minute=30,
    day_of_week='mon-fri'
)

# 每日更新
scheduler.add_job(
    DailyUpdateTask().run_daily_update,
    'cron',
    hour=17,
    minute=0,
    day_of_week='mon-fri'
)

scheduler.start()
```

## 七、备份恢复

### 7.1 数据备份

```bash
#!/bin/bash
# backup.sh - 数据备份脚本

BACKUP_DIR="/backup/xxxcnstock/$(date +%Y%m%d)"
DATA_DIR="./data"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据
tar -czf $BACKUP_DIR/data.tar.gz $DATA_DIR

# 备份配置
cp -r config $BACKUP_DIR/
cp .env $BACKUP_DIR/

# 保留最近30天备份
find /backup/xxxcnstock -type d -mtime +30 -exec rm -rf {} \;

echo "备份完成: $BACKUP_DIR"
```

### 7.2 数据恢复

```bash
#!/bin/bash
# restore.sh - 数据恢复脚本

BACKUP_DATE=$1
BACKUP_DIR="/backup/xxxcnstock/$BACKUP_DATE"

if [ ! -d "$BACKUP_DIR" ]; then
    echo "备份不存在: $BACKUP_DIR"
    exit 1
fi

# 停止服务
# docker-compose down

# 恢复数据
tar -xzf $BACKUP_DIR/data.tar.gz

# 恢复配置
cp -r $BACKUP_DIR/config ./
cp $BACKUP_DIR/.env ./

# 启动服务
# docker-compose up -d

echo "恢复完成"
```

### 7.3 自动备份


## 八、故障处理

### 8.1 常见问题

#### 问题1: 数据获取失败

**现象**: API调用返回错误

**排查**:
```bash
# 检查网络连接
ping api.tushare.pro

# 检查API Token
python -c "import os; print(os.getenv('TUSHARE_TOKEN'))"

# 查看日志
tail -f logs/system/unified_data_service.log
```

**解决**:
- 检查网络连接
- 验证API Token有效性
- 查看Tushare服务状态
- 切换到备用数据源(AKShare)

#### 问题2: 存储空间不足

**现象**: 磁盘空间告警

**排查**:
```bash
# 查看磁盘使用
df -h

# 查看数据目录大小
du -sh data/

# 查看日志大小
du -sh logs/
```

**解决**:
- 清理旧日志
- 压缩历史数据
- 扩展存储空间
- 调整数据保留策略

#### 问题3: 缓存命中率低

**现象**: 查询性能下降

**排查**:
```bash
# 查看缓存统计
python -c "
from services.data_service.storage.optimized_financial_storage import OptimizedFinancialStorageManager
storage = OptimizedFinancialStorageManager()
print(storage.get_cache_stats())
"
```

**解决**:
- 增加缓存容量
- 调整缓存TTL
- 运行数据预热任务
- 检查热点数据

#### 问题4: 更新任务失败

**现象**: 定时任务未执行或失败

**排查**:
```bash
# 查看cron日志
tail -f logs/cron.log

# 手动执行任务
python -m services.data_service.tasks.daily_update_task

# 检查进程
ps aux | grep python
```

**解决**:
- 检查cron配置
- 查看错误日志
- 手动执行任务调试
- 重启调度服务

### 8.2 应急处理

```bash
# 1. 停止所有更新任务
pkill -f "daily_update_task"
pkill -f "incremental_update_task"

# 2. 清理临时文件
rm -rf data/temp/*

# 3. 重启服务
docker-compose restart

# 4. 验证服务状态
docker-compose ps
```
