# 数据服务扩展架构文档

## 1. 项目概述

### 1.1 项目目标

本项目对原有数据服务进行了全面扩展，新增三大核心模块：
- **财务数据模块**：三大财务报表（资产负债表、利润表、现金流量表）
- **市场行为数据模块**：龙虎榜、资金流向、北向资金
- **公告系统模块**：公司公告、重大事项、业绩预告

### 1.2 技术栈

- **后端语言**: Python 3.11+
- **数据处理**: Polars, Pandas, DuckDB
- **存储格式**: Parquet (列式存储)
- **数据源**: Tushare Pro (主), AKShare (备)
- **缓存**: 内存缓存 (LRU + TTL)

## 3. 模块详细设计

### 3.1 财务数据模块

#### 3.1.1 数据模型

**资产负债表 (Balance Sheet)**
```python
{
    "code": "股票代码",
    "report_date": "报告期",
    "total_assets": "总资产",
    "total_liabilities": "总负债",
    "total_equity": "股东权益",
    "current_assets": "流动资产",
    "current_liabilities": "流动负债",
    # ... 其他字段

```

**利润表 (Income Statement)**
```python
{
    "code": "股票代码",
    "report_date": "报告期",

    "operating_cost": "营业成本",
    "gross_profit": "毛利润",
    "net_profit": "净利润",
    # ... 其他字段

```

**现金流量表 (Cash Flow)**
```python
{
    "code": "股票代码",
    "report_date": "报告期",
    "operating_cash_flow": "经营活动现金流",
    "investing_cash_flow": "投资活动现金流",
    "financing_cash_flow": "筹资活动现金流",
    # ... 其他字段
}
```

#### 3.1.2 财务指标计算

| 指标名称 | 计算公式 | 说明 |
|---------|---------|------|
| ROE | 净利润 / 股东权益 | 净资产收益率 |
| ROA | 净利润 / 总资产 | 总资产收益率 |
| 毛利率 | (营业收入-营业成本) / 营业收入 | 盈利能力 |
| 净利率 | 净利润 / 营业收入 | 盈利能力 |
| 资产负债率 | 总负债 / 总资产 | 偿债能力 |
| 流动比率 | 流动资产 / 流动负债 | 短期偿债能力 |
| 营收增长率 | (本期营收-上期营收) / 上期营收 | 成长能力 |
| 经营现金流/净利润 | 经营现金流 / 净利润 | 现金流质量 |

## 4. 存储设计

### 4.1 存储结构

```
data/
├── financial/
│   ├── balance_sheet/          # 资产负债表
│   │   ├── 000001.parquet
│   │   ├── 000002.parquet
│   │   └── ...
│   ├── income_statement/       # 利润表
│   ├── cash_flow/              # 现金流量表
│   └── indicators/             # 财务指标
├── market_behavior/
│   ├── dragon_tiger/           # 龙虎榜数据
│   ├── money_flow/             # 资金流向
│   └── northbound/             # 北向资金
├── announcements/
│   ├── major_events/           # 重大事项
│   ├── performance_forecasts/  # 业绩预告
│   └── equity_changes/         # 股权变动
└── quality_reports/            # 质量报告
```

### 4.2 存储优化

- **Parquet格式**: 列式存储，压缩率高，查询速度快
- **按股票分片**: 每只股票独立文件，便于增量更新
- **Zstd压缩**: 高压缩比，节省存储空间
- **索引缓存**: 内存缓存热点数据，减少IO

## 5. 性能优化

### 5.1 缓存策略

| 缓存类型 | 容量 | TTL | 说明 |
|---------|------|-----|------|
| 财务数据缓存 | 1000条 | 3600s | 热点股票财务数据 |
| 指标缓存 | 1000条 | 3600s | 计算后的财务指标 |
| 代码索引缓存 | 1个 | 300s | 可用股票代码列表 |

### 5.2 并发处理

- **数据获取**: 多线程并发获取多只股票数据
- **数据更新**: 并行处理多只股票更新
- **数据预热**: 并发加载热点数据到内存

### 5.3 增量更新

- **差异检测**: 对比新旧数据，只更新变化部分
- **断点续传**: 支持中断后从上次位置恢复
- **状态持久化**: 更新状态保存到文件

## 6. 监控与告警

### 6.1 监控指标

| 指标 | 说明 | 阈值 |
|------|------|------|
| 数据质量评分 | 综合数据质量 | < 70 告警 |
| 数据新鲜度 | 最后更新时间 | > 24h 告警 |
| 缓存命中率 | 缓存效率 | < 80% 警告 |
| 更新失败率 | 更新任务失败比例 | > 5% 告警 |

### 6.2 告警级别

- **Error**: 数据质量评分<70, 更新延迟>48h
- **Warning**: 数据质量评分<90, 更新延迟>24h, 缓存禁用
