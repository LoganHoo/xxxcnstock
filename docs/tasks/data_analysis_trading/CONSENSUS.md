# 需求共识文档

## 项目目标
本项目旨在构建一个完整的股票分析和交易策略系统，通过数据采集、分析和回测，为用户提供选股建议。

## 核心功能

### 1. 数据采集模块
- **功能**：采集A股历史K线数据（日线级别）
- **范围**：最近3年历史数据
- **更新频率**：每日收盘后更新
- **存储格式**：Parquet格式
- **数据源**：腾讯财经API

### 2. 数据分析模块
- **功能**：基于采集的数据进行多维度分析
- **分析方法**：
  - 因子系统：技术指标、量价指标等
  - 过滤器系统：基本面、流动性、市场、技术等过滤器
  - 模式识别：K线形态、趋势模式等

### 3. 选股模块
- **功能**：基于分析结果选择优质股票
- **策略**：
  - 多因子策略
  - 趋势跟随策略
- **评估**：通过回测验证策略效果

### 4. TDD验证模块
- **功能**：使用测试驱动开发方法验证系统功能
- **测试范围**：
  - 数据采集测试
  - 因子计算测试
  - 过滤器效果测试
  - 策略回测测试

### 5. 报告与缺陷分析模块
- **功能**：生成测试报告，分析系统缺陷
- **报告内容**：
  - 测试结果
  - 性能指标
  - 缺陷统计
  - 改进建议

## 技术栈确认
- **后端**：Python 3.11+ / Flask / SQLAlchemy
- **存储**：MySQL 5.0+ / Redis
- **部署**：Docker / Docker Compose
- **数据处理**：Polars / DuckDB / Parquet

## 系统边界确认
- ✅ **历史行情数据**：存储A股历史K线数据（日线级别）
- ❌ **实时行情数据**：不存储实时行情，仅通过API实时查询
- ✅ **数据更新频率**：每日收盘后更新一次历史数据
- ✅ **数据范围**：最近3年历史K线数据
- ✅ **回测功能**：支持策略回测
- ❌ **实盘交易**：仅做流水线的SOP，不做实盘交易

## 实施计划
1. **数据采集模块**：实现数据采集和存储功能
2. **数据分析模块**：实现因子计算、过滤器和模式识别
3. **选股模块**：实现策略和回测功能
4. **TDD验证模块**：实现测试用例和验证流程
5. **报告与缺陷分析模块**：实现报告生成和缺陷分析

## 质量标准
- **代码规范**：遵循PEP8，使用flake8检查
- **测试覆盖**：核心功能测试覆盖率达到80%以上
- **性能要求**：数据处理速度满足每日更新需求
- **可靠性**：系统稳定运行，数据准确无误
