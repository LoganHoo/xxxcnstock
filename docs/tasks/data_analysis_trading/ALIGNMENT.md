# 需求对齐文档

## 项目概述
本项目是一个股票分析和交易策略系统，核心功能包括数据采集、数据分析、选股和交易。

## 核心需求

### 1. 数据采集
- **目标**：确保采集到有效的历史和实时股票数据
- **范围**：A股历史K线数据（日线级别），最近3年历史数据
- **更新频率**：每日收盘后更新一次历史数据
- **数据格式**：使用Parquet格式存储，高效列式存储

### 2. 数据分析与选股
- **目标**：基于采集的数据进行分析，选择优质股票进行交易
- **分析方法**：使用因子系统、过滤器系统和模式识别进行多维度分析
- **策略**：支持多因子策略和趋势跟随策略
- **评估**：通过回测验证策略效果

### 3. TDD验证与评审
- **目标**：使用测试驱动开发方法验证系统功能
- **测试范围**：数据采集、因子计算、过滤器效果、策略回测
- **评审流程**：定期对系统进行评审，发现并修复缺陷

### 4. 测试报告与缺陷分析
- **目标**：生成详细的测试报告，分析系统缺陷
- **报告内容**：测试结果、性能指标、缺陷统计、改进建议
- **缺陷处理**：跟踪和修复系统缺陷，持续优化系统

## 技术栈
- **后端**：Python 3.11+ / Flask / SQLAlchemy
- **前端**：Next.js 16+ / TypeScript / Tailwind CSS（可选）
- **存储**：MySQL 5.0+ / Redis
- **部署**：Docker / Docker Compose
- **数据处理**：Polars / DuckDB / Parquet

## 系统边界
- ✅ **历史行情数据**：存储A股历史K线数据（日线级别）
- ❌ **实时行情数据**：不存储实时行情，仅通过API实时查询
- ✅ **数据更新频率**：每日收盘后更新一次历史数据
- ✅ **数据范围**：最近3年历史K线数据
- ✅ **回测功能**：支持策略回测
- ❌ **实盘交易**：仅做流水线的SOP，不做实盘交易

## 歧义询问清单

| 序号 | 歧义点 | 详细描述 | 建议解决方案 |
| --- | --- | --- | --- |
| 1 | 数据采集来源 | 系统需要从哪些API获取股票数据？ | 建议使用腾讯财经API或其他可靠的数据源 |
| 2 | 因子系统扩展 | 是否需要扩展因子系统，添加新的因子？ | 建议根据策略需求，定期评估和添加新因子 |
| 3 | 回测参数 | 回测的时间范围和参数设置如何确定？ | 建议使用最近3年数据进行回测，参数根据策略特性调整 |
