# 数据人工验证系统设计

**日期**: 2026-04-03
**状态**: 已批准

---

## 1. 概述

每天晚上数据采集和自动质量检查完成后，通过人工验证流程确保数据准确性。

**核心目标**:
- 验证K线数据连续性、最新日期
- 随机抽检股票数据供人工核对
- 第三方数据源交叉验证

---

## 2. 流程设计

```
每日收盘后(18:00-20:00)
    ↓
数据采集任务 (data_collect.py)
    ↓
自动质量检查 (data_audit.py)
    ↓
人工验证流程 (NEW: manual_verification.py)
    ↓
HTML报告生成
    ↓
邮件通知 (287363@qq.com)
```

---

## 3. 功能模块

### 3.1 连续性检查
- 验证K线数据日期连续、无缺失
- 检查标准：每个股票相邻交易日应该连续
- 异常告警：跳过节假日导致的缺失是正常的

### 3.2 最新数据检查
- 验证最新日期=今日
- 验证数据完整性：开盘、收盘、最高、最低、成交量都有值
- 异常告警：数据缺失或日期不对

### 3.3 人工抽检(3个股票)
- 随机抽取3个股票代码
- 展示数据表格供人工核对
- 人工确认后记录验证结果

### 3.4 第三方验证(1个股票)
- 调用AKShare或东方财富API
- 对比内部数据与第三方数据
- 差异超过阈值(0.1%)则告警

## 6. 定时任务配置

在 `cron_tasks.yaml` 中添加:

```yaml
manual_verification:
  command: python scripts/manual_verification.py
  schedule: "0 20 * * *"  # 每天20:00执行
  description: 人工数据验证
  timeout: 1800
```
