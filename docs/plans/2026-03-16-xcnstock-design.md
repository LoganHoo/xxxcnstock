# XCNStock 股票分析系统设计文档

## 概述

基于第一性原理构建的A股专业交易系统，集成选股分析、打板策略、智能通知于一体的微服务架构。

## 设计目标

- **专业性**: 面向专业交易，高可靠性、实时性
- **智能化**: 多维度选股 + 打板策略分析
- **自动化**: 后台服务自动运行，智能推送通知
- **可扩展**: 微服务架构，模块解耦，易于扩展

## 二、选股服务 (Stock Service)

### 四维度筛选策略

#### 1. 基本面筛选 (权重35%)

| 指标 | 条件 | 权重 |
|------|------|------|
| PE市盈率 | 0 < PE < 50 | 15% |
| PB市净率 | 0 < PB < 10 | 10% |
| ROE | > 10% | 20% |
| 营收增长率 | > 15% | 15% |
| 净利润增长率 | > 10% | 20% |
| 负债率 | < 60% | 20% |

#### 2. 量价分析 (权重25%)

| 指标 | 条件 | 权重 |
|------|------|------|
| 成交量突破 | Vol > MA5_Vol * 1.5 | 30% |
| 价格突破 | Close > MA20 | 25% |
| RSI | 40 < RSI < 70 | 20% |
| MACD | DIF > DEA | 15% |
| 布林带 | Close > MID | 10% |

#### 3. 资金流向 (权重25%)

| 指标 | 条件 | 权重 |
|------|------|------|
| 主力净流入 | > 0 | 35% |
| 北向资金 | 连续3日流入 | 25% |
| 大单净比 | > 0 | 20% |
| 散户离场 | 散户占比下降 | 20% |

#### 4. 情绪面分析 (权重15%)

| 指标 | 条件 | 权重 |
|------|------|------|
| 板块热度 | 所属板块涨幅前5 | 35% |
| 个股热度 | 换手率 3%~15% | 25% |
| 连板数 | 首板优先 | 20% |
| 市场情绪 | 大盘上涨 | 20% |

### 综合评分公式

```
总分 = 基本面分数 × 0.35 + 量价分数 × 0.25 + 资金分数 × 0.25 + 情绪分数 × 0.15
```

## 三、打板服务 (Limit Service)

### 涨停预判算法

**第一性原理**: 涨停 = 供需失衡 + 情绪共振

```python
def predict_limit_up(stock_data):
    factors = {
        "price_momentum": {
            "涨幅": stock_data["change_pct"],
            "距涨停": (10 - stock_data["change_pct"]) / 10,
            "权重": 0.30
        },
        "volume_energy": {
            "量比": stock_data["volume_ratio"],
            "换手率": stock_data["turnover_rate"],
            "权重": 0.25
        },
        "seal_strength": {
            "封单金额": stock_data.get("seal_amount", 0),
            "封单/流通市值": stock_data.get("seal_ratio", 0),
            "权重": 0.25
        },
        "sector_effect": {
            "板块涨幅": stock_data["sector_change"],
            "板块内涨停数": stock_data["sector_limit_count"],
            "权重": 0.20
        }
    }
    return calculate_limit_probability(factors)
```

### 封板强度评估

| 指标 | 强势封板 | 中等封板 | 弱势封板 |
|------|---------|---------|---------|
| 封单金额 | > 流通市值 5% | 1-5% | < 1% |
| 封板时间 | 10:00前 | 10:00-14:00 | 14:00后 |
| 开板次数 | 0次 | 1-2次 | > 2次 |
| 封单变化 | 持续增加 | 基本稳定 | 持续减少 |
| 板块地位 | 龙头股 | 跟风股 | 补涨股 |

### 次日预判模型

| 今日特征 | 次日预期 | 概率 | 操作建议 |
|---------|---------|------|---------|
| 强势首板 + 龙头 + 热门题材 | 一字板/高开涨停 | 70% | 竞价可参与 |
| 强势首板 + 跟风 + 热门题材 | 高开冲高 | 60% | 竞价观望 |
| 首板 + 尾盘封板 | 平开/低开震荡 | 50% | 谨慎参与 |
| 连板 + 放量 | 高开分歧 | 55% | 观察分歧转一致 |
| 连板 + 缩量 | 高开加速 | 65% | 竞价可博弈 |

### 信号等级

| 等级 | 条件 | 通知级别 |
|------|------|---------|
| S级 | 龙头 + 首板 + 强封 + 热门题材 | 即时推送 + 语音提醒 |
| A级 | 龙头/跟风 + 首板 + 中强封 | 即时推送 |
| B级 | 跟风 + 首板/连板 + 中等封板 | 批量推送 |
| C级 | 补涨股 + 弱封板 | 每日汇总 |

## 四、通知服务 (Notify Service)

### 通知渠道

| 渠道 | 类型 | 场景 |
|------|------|------|
| 微信 | Server酱 | 即时通知 |
| 钉钉 | 机器人 | 即时通知 |
| 邮件 | SMTP | 详细报告 |
| Webhook | 自定义 | 系统集成 |

### 通知策略

| 信号等级 | 微信 | 钉钉 | 邮件 | 日志 |
|---------|------|------|------|------|
| S级 | 即时 | 即时 | 即时 | 详细 |
| A级 | 即时 | 即时 | 即时 | 详细 |
| B级 | 批量 | 批量 | 批量汇总 | 详细 |
| C级 | 每日 | - | 每日汇总 | 详细 |

### 频率控制

- 同一股票通知间隔: 5分钟
- 同等级每日上限: 50条
- 批量通知间隔: 15分钟
- 静默时段: 23:00 - 07:00

### 日志系统

| 日志类型 | 存储路径 | 格式 | 保留周期 |
|---------|---------|------|---------|
| 系统日志 | `logs/system/` | 文本 | 30天 |
| 信号日志 | `logs/signals/` | JSON | 180天 |
| 交易日志 | `logs/trades/` | JSON | 365天 |
| 告警日志 | `logs/alerts/` | JSON | 90天 |
