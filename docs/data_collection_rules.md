# K线数据采集规则文档

## 核心铁律

**交易日盘中（9:30-15:00）禁止采集当日K线数据**

---

## 一、采集时间规则

### 1.1 允许采集的时段

| 时段 | 可采集数据类型 | 说明 |
|------|----------------|------|
| 交易日 15:30 后 | 当日数据 | 收盘后数据已稳定，正常采集 |
| 非交易日任何时间 | 历史数据 | 周末/节假日可补采历史数据 |
| 任何时间（带--date） | 指定历史日期 | 强制指定历史日期不受时间限制 |

### 1.2 禁止采集的时段

| 时段 | 禁止原因 | 后果 |
|------|----------|------|
| 交易日 9:30-15:00 | 盘中行情不完整 | 收盘价、成交量不准确 |
| 交易日 15:00-15:30 | 数据可能未稳定 | 建议等待30分钟缓冲期 |

---

## 二、关键区分原则

### 2.1 当日数据 vs 历史数据

```
当日数据：数据日期 == 今天
  → 必须收盘后采集（15:30+）
  → 盘中采集会触发强制退出

历史数据：数据日期 < 今天
  → 任何时间都可采集
  → 不受交易日/盘中限制
```

### 2.2 场景示例

| 场景 | 当前时间 | 目标数据日期 | 是否允许 | 说明 |
|------|----------|--------------|----------|------|
| 周五盘中采集周五数据 | 周五 10:00 | 2026-04-17 | ❌ 禁止 | 盘中数据不完整 |
| 周五收盘后采集周五数据 | 周五 15:30 | 2026-04-17 | ✅ 允许 | 正常采集 |
| 周六采集周五数据 | 周六 10:00 | 2026-04-17 | ✅ 允许 | 非交易日可采历史 |
| 盘中强制补采上周数据 | 周五 10:00 | 2026-04-10 | ✅ 允许 | 历史数据不受限制 |

## 三、技术实现

### 3.1 核心模块

**文件**: `core/market_guardian.py`

```python
# 当日数据检查（今天 = 2026-04-18）
enforce_market_closed(target_date="2026-04-18")
# 如果是交易日盘中 → sys.exit(1) 强制退出

# 历史数据检查（昨天 = 2026-04-17）
enforce_market_closed(target_date="2026-04-17")
# 任何时间都允许 → 正常执行
```

### 3.2 采集脚本

**文件**: `scripts/pipeline/data_collect.py`

```python
# K线数据采集时间检查
if self.force_date:
    # 强制模式：采集指定历史日期，不受时间限制
    logger.info(f"强制重新采集历史数据: {self.force_date}")
else:
    # 正常模式：检查是否允许采集当日数据
    # 交易日盘中会强制退出！
    today = datetime.now().strftime('%Y-%m-%d')
    enforce_market_closed(target_date=today)
```

## 四、定时任务配置

### 4.1 推荐执行时间

**文件**: `config/cron_tasks.yaml`

```yaml
# 交易日盘后采集（16:00）
- name: "data_fetch"
  schedule: "0 16 * * 1-5"  # 交易日 16:00
  script: "scripts/pipeline/data_collect.py"
  
# 断点续传（16:30）
- name: "data_fetch_retry"
  schedule: "30 16 * * 1-5"
  script: "scripts/pipeline/data_collect.py --retry"
```

### 4.2 任务执行流程

```
16:00 - 基础数据采集（收盘后）
  ↓
16:30 - 断点续传（补充缺失数据）
  ↓
16:50 - 数据审计（验证完整性）
  ↓
17:15 - 指标计算（CVD等）
  ↓
17:30 - 复盘分析
```

## 五、违规后果

### 5.1 数据质量问题

- 盘中采集的K线数据收盘价不准确
- 成交量数据不完整
- 导致技术指标计算错误

### 5.2 业务影响

- 选股策略基于错误数据产生误导信号
- 复盘报告数据不可靠
- **严重级别：致命**

### 5.3 防护措施

1. **启动时强制检查** - 盘中采集当日数据直接 `sys.exit(1)`
2. **写入前守卫检查** - 检测 fetch_time，盘中数据拒绝写入
3. **定时任务配置** - 15:30 后执行，兜底防护

## 六、异常处理

### 6.1 盘中误启动

```
[2026-04-17 10:30:00] 🚨 交易日盘中 (10:30)，禁止采集当日数据！
根据项目规则：交易日盘中（9:30-15:00）禁止采集当日K线数据
原因：盘中行情不完整，采集会导致数据污染

如需补采历史数据，请使用 --date 参数指定历史日期
示例: python data_collect.py --date 2026-04-15

程序已强制终止
```

### 6.2 历史数据补采

```bash
# 允许：盘中补采历史数据
python scripts/pipeline/data_collect.py --date 2026-04-10

# 允许：非交易日补采
python scripts/pipeline/data_collect.py
```
