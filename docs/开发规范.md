## 一、代码规范

遵循 **PEP 8** 规范，使用以下工具进行检查：

```bash

# 类型检查
mypy services/ filters/ core/

# 代码格式化
black services/ filters/ core/
```

### 1.2 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 模块 | 小写+下划线 | `balance_sheet.py` |
| 类 | 大驼峰 | `BalanceSheetFetcher` |
| 函数 | 小写+下划线 | `fetch_balance_sheet()` |
| 常量 | 大写+下划线 | `MAX_RETRY_COUNT` |
| 变量 | 小写+下划线 | `stock_code` |
| 私有 | 前缀下划线 | `_internal_method()` |

def fetch_with_retry(code: str, max_retry: int = 3) -> pd.DataFrame:
    """带重试的数据获取"""
    for attempt in range(max_retry):
        try:
            data = _fetch_data(code)

            return data

## 二、Git工作流程

### 2.1 分支模型

```
main (生产分支)
  │
  ├── develop (开发分支)
  │     │
  │     ├── feature/phase-1-financial (功能分支)
  │     ├── feature/phase-2-market-behavior
  │     ├── feature/phase-3-announcement
  │     └── ...
  │
  ├── release/v1.0 (发布分支)
  │
  └── hotfix/fix-bug (热修复分支)
```

### 2.2 提交规范

提交信息格式：

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type类型**：

| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | 修复bug |
| docs | 文档更新 |
| style | 代码格式调整 |
| refactor | 重构 |
| perf | 性能优化 |
| test | 测试相关 |
| chore | 构建/工具相关 |

**示例**：

```bash
feat(financial): 添加资产负债表获取器

- 实现BalanceSheetFetcher类
- 支持Tushare和AKShare双数据源
- 添加数据验证和错误处理

Closes #123
```

### 2.3 开发流程

```bash
# 1. 创建功能分支
git checkout -b feature/phase-1-financial develop

# 2. 开发并提交
git add .
git commit -m "feat(financial): 添加资产负债表获取器"

# 3. 保持与develop同步
git fetch origin
git rebase origin/develop

# 4. 推送分支
git push origin feature/phase-1-financial

# 5. 创建Pull Request
# 在GitHub/GitLab上创建PR，请求合并到develop

# 6. 代码审查通过后合并
# 由维护者合并到develop分支
```

### 3.1 代码文档

所有公共类和方法必须有文档字符串：

```python
def calculate_roe(
    net_profit: float,
    equity: float
) -> float:
    """
    计算净资产收益率(ROE)
    
    ROE = 净利润 / 股东权益
    
    Args:
        net_profit: 净利润
        equity: 股东权益
    
    Returns:
        ROE值，百分比形式
    
    Raises:
        ValueError: 股东权益为0时抛出
    
    Examples:
        >>> calculate_roe(100, 1000)
        10.0
    """
    if equity == 0:
        raise ValueError("股东权益不能为0")
    return (net_profit / equity) * 100
```

## 七、发布规范

### 7.1 版本号规则

遵循语义化版本：

```
主版本号.次版本号.修订号
   │       │       │
   │       │       └── 修复bug
   │       └────────── 新功能
   └────────────────── 不兼容修改
```

### 7.2 发布流程

```bash
# 1. 更新版本号
# 更新 __version__ = "1.0.0"

# 2. 更新CHANGELOG.md

# 3. 创建发布分支
git checkout -b release/v1.0 develop

# 4. 修复bug并提交

# 5. 合并到main
git checkout main
git merge --no-ff release/v1.0

# 6. 打标签
git tag -a v1.0.0 -m "Release version 1.0.0"

# 7. 合并回develop
git checkout develop
git merge --no-ff release/v1.0

# 8. 删除发布分支
git branch -d release/v1.0
```
